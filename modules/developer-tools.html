Get up and running with client credentials, API specifications, and integration tools

Get access to code snippets, use cases, and examples to learn about the ESW APIs and our plugins to integrate with our Checkout, Shipping, and Returns solutions.

Get Started Integrating with ESW involves three steps: Obtaining client credentials so that ESW can authenticate your API calls. Generating an API client library from Swagger or Postman collections so that your system can interact with the ESW APIs. Making a test API call to confirm that everything is working as expected.
looks_one

Obtain Client Credentials

ESW uses the Security Token Service (STS) platform to issue, validate, and renew security tokens that are required to authenticate with ESW APIs and access API endpoints.
looks_two

Generate an API client

ESW APIs are built with OpenAPI specification. You can generate client SDK on the fly. Alternatively, you can use our extensive Postman collections to generate sample code snippets.
looks_3

Make a test API request

To confirm that your ESW integration is working as expected, make a test API request using your test client credentials to try our Checkout. You can also use our demo client credentials.
ESW APIs Interact with ESW APIs to bootstrap sample projects, send test mode API requests, and debug your integration. You can integrate with the ESW Checkout, Shipping, and Returns solutions using the APIs.
